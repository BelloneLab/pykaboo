# Python dependencies for PyKaboo (CamApp Live Detection)
# ---------------------------------------------------------------------------
# Quick start (CPU/GPU auto):
#     pip install -r requirements.txt
#
# For NVIDIA GPU acceleration (recommended — live mask+pose needs CUDA), install
# the CUDA build of PyTorch FIRST, then the rest:
#     pip install torch==2.12.0 torchvision==0.27.0 --index-url https://download.pytorch.org/whl/cu126
#     pip install -r requirements.txt
#
# External, NOT installed by pip:
#   * FFmpeg            — required for video recording. Install separately and
#                         put ffmpeg/ffprobe on PATH (e.g. `winget install Gyan.FFmpeg`).
#   * FLIR Spinnaker    — install the vendor PySpin wheel that ships in ./PySpin
#                         (matches the Spinnaker SDK + your Python version), e.g.
#                         pip install "PySpin/spinnaker_python-3.2.0.62-cp310-cp310-win_amd64/spinnaker_python-3.2.0.62-cp310-cp310-win_amd64.whl"
#
# Bundled models: prebuilt TensorRT engines ship under pykaboo/models/ and the
# Live Detection panel points at them automatically. The full (portable) weights
# are too big for git, so they are GitHub Release assets — fetch them with
#     python scripts/download_models.py
# then rebuild engines for your machine (engines are GPU/TensorRT-specific). See
# pykaboo/models/README.md.
# ---------------------------------------------------------------------------

# name discovery (pyside6-essentials alone is NOT enough).

# Deep learning (install the cu126 build of torch/torchvision for GPU — see header)
torch>=2.2

# TensorRT acceleration (NVIDIA GPU only) — powers the "Max GPU (TensorRT)" GPU
# mode for RF-DETR seg + YOLO pose (~2x faster than torch fp16). Use the CUDA-12
# build to match the cu126 torch wheels above; the plain `tensorrt` package pulls
# a CUDA-13 build that will NOT load alongside cu126. onnx + onnxslim are required
# to *build* engines. Engines are hardware/driver/TensorRT-version specific:
# rebuild on the target machine with
#     python scripts/build_rfdetr_engine.py --checkpoint <seg.pth>
#     python scripts/build_yolo_pose_engine.py --pose <pose.pt>
# The app runs fine without TensorRT (it falls back to the torch GPU path).
tensorrt-cu12==10.13.3.9.post1
onnx>=1.16
onnxslim>=0.1.71
